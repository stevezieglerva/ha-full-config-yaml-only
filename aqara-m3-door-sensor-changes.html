Conversion recipe — device block → state

Replace each platform: device block with the matching state form. Keep any id: that was present.

type: opened trigger

- type: opened
  platform: device
  device_id: 476a681c…
  entity_id: 638bc547…
  domain: binary_sensor
- platform: state
  entity_id: <NEW_ENTITY>
  from: 'off'
  to: 'on'
  # id: small  # keep if present

type: not_opened trigger

- type: not_opened
  platform: device
  device_id: 476a681c…
  …
- platform: state
  entity_id: <NEW_ENTITY>
  from: 'on'
  to: 'off'
  # id: small  # keep if present

type: is_open condition

- type: is_open
  condition: device
  device_id: 476a681c…
  entity_id: 638bc547…
  domain: binary_sensor
- condition: state
  entity_id: <NEW_ENTITY>
  state: 'on'
Unavailable-safe Explicit from/to on triggers won't misfire when the Matter entity drops to unavailable; a state: 'on' condition correctly reads unavailable as "not open."
Schema note These blocks still use classic platform: state / condition: state, which remains valid in 2026.7. If you prefer to match the newer style HA wrote in commit 277656e, the modern keys are trigger: state (under triggers:) and action: — both forms work; don't mix keys within one block.
